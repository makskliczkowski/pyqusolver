## 2024-05-18 - Avoid redundant loop passes in Numba when out buffers are provided
**Learning:** In Numba JIT functions, explicit two-pass loops (count, allocate, populate) are surprisingly efficient and often faster than Numba's `np.nonzero` for creating new arrays. However, if an output buffer (`out`) is already pre-allocated, performing the initial counting pass is completely redundant.
**Action:** When a pre-allocated `out` buffer is provided to a Numba JIT function, always use a single-pass loop to directly populate `out` and return a slice (`out[:k]`) instead of doing a double-pass to count elements first. This yields significant performance improvements (e.g. ~37% faster for large arrays).
